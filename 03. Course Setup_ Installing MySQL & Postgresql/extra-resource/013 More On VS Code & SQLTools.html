In this course, we'll use VS Code with the SQLTools extension for interacting with the database, since this allows us to use a uniform interface and save queries into (re-executable) text files (.sql files) with ease.

Of course, you could use any other client (e.g. the clients shown earlier in this section) as well.

If you are using VS Code with SQLTools, we strongly recommend that you do explore the VS Code settings for this extension:

  1. Open the "User Settings" (global settings) or "Workspace Settings" (project-specific settings) in VS Code (More Details)

  2. Search for "SQLTools"

  3. Browse the SQLTools settings and adjust them to best fit your needs

There's one specific setting that you might want to turn off: "Auto Open Session Files". If that's turned on, a new .sql file will be opened automatically, every time you switch the database connection. As this can become annoying, you might want to disable this behavior with the mentioned setting.
